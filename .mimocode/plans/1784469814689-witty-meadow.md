# Gebya — Field Test Readiness (5-10 Merchants)

## The Only Question That Matters Right Now

Do 5-10 real shop owners actually use this instead of their notebook?

Everything below is scoped to de-risking that field test. No refactor work, no scope creep.

---

## Before Field Test — Do Now (4 items, < 1 day total)

### 1. Rotate Exposed Secrets
**Why**: Real merchants' real phone numbers go through the SMS key during the test. Don't expose test participants' data.
- **File**: `.env` — rotate `SMS_ETHIOPIA_API_KEY` and `VAPID_PRIVATE_KEY`
- Move both to Vercel environment variables
- Replace `.env` values with placeholder strings
- **Time**: 1 hour

### 2. Fix OTP to Use Cryptographic Random
**Why**: Protects the phone-based onboarding flow you're about to actually use.
- **File**: `artifacts/api-server/src/routes/auth.ts:27-29`
- Change `Math.random()` → `crypto.randomInt(100000, 999999)`
- **Time**: 30 min

### 3. Make Telegram Webhook Secret Required
**Why**: If unset, anyone can POST fake messages to your bot. Telegram is your notification channel for real merchants.
- **File**: `artifacts/api-server/src/routes/telegram.ts:558-562`
- Reject webhook requests if `TELEGRAM_WEBHOOK_SECRET` is not set
- **Time**: 30 min

### 4. Fix Amharic Toast Typo
**Why**: Visible to real Amharic-speaking merchants hitting a transaction limit.
- **File**: `artifacts/gebya/src/components/AppShell.jsx` (search for "ouis")
- Fix the string — do NOT extract to a translation system (still deferred)
- **Time**: 15 min

## What NOT to Do Before Field Test

These are real technical debt. They are all distractions from the field test:

- AppShell.jsx decomposition (1200+ lines) — doesn't get a single additional observation
- Dual schema unification — post-field-test
- JWT refresh tokens / auth rate limiting — 5-10 known users isn't the threat model
- CSP headers — defer to funder demo gate
- CSRF protection — Bearer tokens already provide CSRF protection
- Backup encryption — defer
- Frontend input validation hardening — defer
- Repo hygiene (_triage-* directories) — defer
- Chunk-reload race condition — defer
- Analytics encryption — defer

---

## Field Test Observations to Capture

While merchants use the product, watch for:

1. **First transaction time** — can they create a sale in < 30 seconds?
2. **Credit flow friction** — does the Merro workflow make sense to them?
3. **Offline behavior** — does the data-persists-on-phone trust message land?
4. **Telegram link adoption** — do customers actually connect via Telegram?
5. **Language switching** — do they toggle Amharic, or stay in English?
6. **Error recovery** — when something breaks, can they recover without help?

---

## Post-Field-Test Gate

After 5-10 merchants have used it for 1+ week:
- Debrief: what worked, what didn't, what's missing?
- Then decide: feature gaps vs technical debt vs launch readiness
