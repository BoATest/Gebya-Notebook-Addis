# Fix: Add comma separator to expense page amount inputs

## Problem

Two amount input fields on the expense page don't show comma separators when typing, unlike the sale form and other inputs throughout the app.

## Root Cause

The app has a standard pattern for comma-separated inputs: `fmtInput()` for display + `handleNumericInput` for onChange. Two expense inputs skip this pattern:

1. **TransactionForm.jsx line 508** - Main expense amount input uses `value={amountDisplay}` (raw number) instead of `value={fmtInput(amountDisplay)}`. The sale form (line 301) correctly uses `fmtInput()`.
2. **RecurringExpensesPanel.jsx line 83** - Uses `type="number"` which doesn't support comma display. Needs `type="text"` with `fmtInput()`.

## Changes

### 1. `artifacts/gebya/src/components/TransactionForm.jsx`

**Line 508** - Add `fmtInput()` to display value:
```
- value={amountDisplay}
+ value={fmtInput(amountDisplay || amount)}
```

The `fmtInput` import already exists at line 32. The onChange handler already strips commas correctly.

### 2. `artifacts/gebya/src/components/settings/RecurringExpensesPanel.jsx`

- Import `fmtInput` and `parseInput` from `../../utils/numformat`
- Line 83: Change `type="number"` to `type="text"`
- Line 85: Change `value={reAmount}` to `value={fmtInput(reAmount)}`
- Line 86: Update onChange to strip commas: `e => setReAmount(e.target.value.replace(/,/g, '').replace(/[^\d.]/g, ''))`
- Line 119: Change `parseFloat(reAmount)` to `parseFloat(parseInput(reAmount))`
- Line 18: Change `parseFloat(reAmount)` to `parseFloat(parseInput(reAmount))`
