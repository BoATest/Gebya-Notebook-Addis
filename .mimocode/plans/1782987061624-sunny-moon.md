# Credit Page UI Fixes

## Issues & Fixes

### 1. Remove "+" quick-credit button from customer list rows
**File:** `artifacts/gebya/src/components/CustomerList.jsx` (lines 561-578)
**Problem:** A green/red "+" button appears next to each customer's balance, labeled "Quick add credit." It's unclear and clutters the row.
**Fix:** Remove the entire `{hasBalance && onQuickCredit && (...)}` block. Keep the balance + "birr" label.

### 2. Remove duplicate "Add Customer" button from credit page
**File:** `artifacts/gebya/src/components/CustomerList.jsx`
**Problem:** The CustomerList hero card does NOT have an Add button (good). But the empty state has "Add your first customer" (fine). The action bar at bottom (App.jsx:3257-3278) is the only "Add Customer" button. So issue #2 might be about a different screen — possibly the SupplierList hero card's "Add" button appearing on the Suppliers view. Let me verify:
- CustomerList hero card: NO Add button → ✓
- Bottom action bar: "Add Customer" (blue) → only when `creditView === 'customers'` and no customer selected → ✓ (this is the one to keep)
- SupplierList hero card: Has "Add" button (red) inside the hero → this is a second "Add Supplier" entry point
- Bottom action bar: Currently shows "+ Add Customer" even on supplier view → BUG

**Fix in App.jsx (line 3258-3278):**
- When `creditView === 'customers'`: show "+ Add Customer" (blue)
- When `creditView === 'suppliers'`: show "+ Add Supplier" (red)

### 3. Button label and color should change based on creditView tab
**File:** `artifacts/gebya/src/App.jsx` (lines 3257-3278)
**Problem:** The bottom action bar always says "+ Add Customer" regardless of whether Customers or Suppliers tab is active.
**Fix:** Conditionally render based on `creditView`:
- Customers tab: blue button, "Add Customer", opens CustomerForm
- Suppliers tab: red button, "Add Supplier", opens SupplierForm

### 4. Add action buttons to SupplierDetail (Add Purchase + Pay Supplier)
**File:** `artifacts/gebya/src/components/SupplierDetail.jsx`
**Problem:** SupplierDetail receives `onAddPurchase` and `onPaySupplier` props (from App.jsx:3028-3035) but renders NO action buttons to use them. CustomerDetail has a full action bar with "You Gave (Dubie)" + "You Got (Paid)". SupplierDetail needs equivalent buttons.
**Fix:** Add a bottom action bar to SupplierDetail (after the trust line) with two buttons:
- "PURCHASE (+)" — red, opens purchase form (supplier owes more)
- "PAY (-)" — green, opens payment form (reduce what you owe)

This mirrors CustomerDetail's action bar but with reversed semantics (supplier: you owe, customer: they owe).

### 5. Remove duplicate Add Supplier button from SupplierList hero card
**File:** `artifacts/gebya/src/components/SupplierList.jsx` (lines 161-178)
**Problem:** The SupplierList hero card has an "Add" button AND the bottom action bar will also have "Add Supplier" (after fix #3). Two entry points.
**Fix:** Remove the "Add" button from the SupplierList hero card. The bottom action bar is the single consistent entry point (same pattern as customers).
