# Plan: Add Payment Method Selection to Credit Payment Receive

## Context
When a shopkeeper records a payment (collecting money from a customer who owes), there's no way to indicate **how** the payment was received — cash, bank transfer, or mobile wallet. The sales recording flow (`TransactionForm` + `PaymentTypeChips`) already supports this via user-configured banks and wallets from Settings. We need to bring the same capability to the credit payment receive flow.

## Current State
- **Sales recording** (`TransactionForm.jsx`): Uses `PaymentTypeChips` with `enabledProviders` (banks/wallets from settings). Stores `payment_type` + `payment_provider` on `transactions`.
- **Credit payment receive** (`CustomerTransactionSheet.jsx`): Only records amount + item_note. No payment method tracking. The `customer_transactions` DB table has no `payment_method` column.
- **Payment channels** (`paymentChannels.js`): Full infrastructure for banks/wallets, already loaded as `enabledProviders` in App.jsx state.

## Changes

### 1. DB Schema — `artifacts/gebya/src/db.js`
Add `payment_method` and `payment_provider` to `customer_transactions` in a new version (v19):
```
customer_transactions: '++id, customer_id, type, amount, due_date, payment_method, payment_provider, ...'
```

### 2. Normalizer — `artifacts/gebya/src/utils/customerLedgerMutations.js`
Include `payment_method` and `payment_provider` in `normalizeCustomerTransactionDraft()` return value:
- `payment_method`: `'cash' | 'bank' | 'wallet'` (from payload)
- `payment_provider`: string or null (bank/wallet name from payload)

### 3. Pass `enabledProviders` to `CustomerTransactionSheet`
- **`App.jsx`** (~line 3433): Add `enabledProviders={enabledProviders}` prop to both `<CustomerTransactionSheet>` usages (new + edit).
- **`CustomerTransactionSheet.jsx`**: Accept `enabledProviders` prop.

### 4. Add Payment Method Chips UI — `CustomerTransactionSheet.jsx`
- Add state: `const [paymentMethod, setPaymentMethod] = useState('cash')` and `const [paymentProvider, setPaymentProvider] = useState('')`.
- In edit mode, pre-fill from `editingTransaction.payment_method` / `editingTransaction.payment_provider`.
- Build payment options list from `enabledProviders` (same pattern as `PaymentTypeChips`): Cash + enabled banks + enabled wallets.
- Render chip selection UI **only when `isPayment`** (not for credit_add), placed after the amount input and before the note field.
- Include `payment_method` and `payment_provider` in the `onSave` payload.

### 5. Save Handler — `App.jsx`
In `handleSaveCustomerTransaction` (~line 2295): spread `payment_method` and `payment_provider` from the payload into the `entry` object written to DB.
