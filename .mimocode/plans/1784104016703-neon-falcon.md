# Report Page Enhancement Plan

## Current State Analysis

The report page (`artifacts/gebya/src/components/ReportView.jsx`, 1445 lines) is a merchant-facing "Shop Check" screen for Ethiopian shopkeepers. It is the third tab in the bottom nav (labeled "Report" / "ሪፖርት").

### What Currently Exists

| Section | What it shows |
|---------|---------------|
| Header | Shop name, Ethiopian date, privacy toggle |
| Insight Strip | Horizontal bar: today's staff/sales/credits/transfers counts |
| Time Range | Sticky tabs: Today, Week, Month, Custom (with date pickers) |
| KPI Grid (6 cards) | Sold, Spent, Collected, Cash to Expect, Transfer Expected, Total Loan Given |
| Search | Text search across transactions |
| Filter | By staff member/seller |
| Staff Sales Today | List of staff + their sales |
| Owner Alerts | High-value sales + overdue credit flags |
| Reminder Summary | Overdue count, total owed, reminded count |
| Cash Closing | Actual vs expected cash/transfer reconciliation |
| Recent Transactions | Last 3 transactions |
| Full History | All transactions (expand/collapse) |
| Export | CSV download (behind `advanced_reports` entitlement) |
| Bottom Action Bar | Filter, Export, History buttons |

### What's Missing (compared to a rich product)

1. **No visual charts** — the entire page is text/numbers only. A merchant can't *see* trends.
2. **No profit metric** — requires mental math across 3 separate KPI cards.
3. **No period comparison** — no week-over-week, month-over-month deltas.
4. **No item/category breakdown** — can't tell which products drive revenue or where money goes.
5. **No daily breakdown** — week/month views show a flat total, not daily patterns.
6. **No payment method visualization** — cash vs mobile/bank split is just two numbers.
7. **No credit health score** — on-time rate exists in `customerMetrics.js` but isn't surfaced here.
8. **No shareable visual report** — CSV only, no image/PDF export.

### Key Discoveries

- **Profit data already exists** in IndexedDB: every transaction row has `cost_price`, `quantity`, and `profit` fields. None of these feed into the report.
- **Credit analytics pipeline** in `customerMetrics.js` (561 lines) has FIFO allocation, on-time rates, credit grades, and PDF export — barely used on the report page.
- **Entitlement gate** `hasEntitlement('advanced_reports')` is already wired at line 713. Premium features can be gated without new infrastructure.
- **No chart library** in `package.json`. Any visualization must be pure CSS/SVG (keeps the PWA light for low-end Android phones).

## Improvement Plan

### Phase 1 — Quick Wins (1-2 weeks, no new dependencies)

#### 1. Net Profit KPI Card
**What:** Add a 7th KPI card showing `Sold - Spent - Loan Given` for the selected period.
**Why:** This is the single number a shopkeeper cares about most. Currently requires mental math.
**Data:** `metrics.totalSold`, `metrics.spentToday`, `metrics.totalLoanGiven` — all already computed in `computeReportMetrics()`.
**Files:** `reportSelectors.js` (add `netProfit` to return), `ReportView.jsx` (add one `<SummaryCard>`).
**Color:** Green when positive, red when negative. Show "—" when profit data is unavailable.

#### 2. Period Comparison Delta Badges
**What:** When viewing Week or Month, show small pill badges on each KPI: "↑12%" green or "↓8%" red.
**Why:** Merchants need to know if business is improving or declining vs the prior equivalent period.
**Data:** New `computePeriodComparison()` function that runs `buildReportRows` + `computeReportMetrics` for the preceding identical-length window.
**Files:** `reportSelectors.js` (new function), `ReportView.jsx` (conditional badges in `<SummaryCard>`).
**Note:** Skip for "today" tab. Memoize aggressively — two extra computation calls on every render.

#### 3. Expense Category Breakdown
**What:** Collapsible section showing top 5 expense categories with horizontal bar visualization.
**Why:** Answers "where did my money go?" — currently unanswerable.
**Data:** Group `expenseRows` by `item_note || item_name || 'Other'`, sum amounts, compute percentages.
**Files:** `reportSelectors.js` (new `computeExpenseBreakdown`), `ReportView.jsx` (new collapsible `<Section>`).

#### 4. Top-Selling Items
**What:** Collapsible section showing top 5 items by revenue with quantity + total birr.
**Why:** Surfaces which products drive the business.
**Data:** Group `saleRows` by `item_name || item_note`, aggregate quantity + revenue, rank by revenue.
**Files:** `reportSelectors.js` (new `computeTopItems`), `ReportView.jsx` (new collapsible `<Section>`).

#### 5. Enhanced Credit Health Row
**What:** Add a 4th cell to the existing `ReminderSummaryCard` showing on-time payment rate %.
**Why:** Gives a single-glance credit health snapshot without visiting the credit page.
**Data:** `shopOnTimeRate()` from `customerMetrics.js` — the prop `enrichedCustomerSummaries` is already passed to `ReportView`.
**Files:** `ReportView.jsx` (add one grid cell to `ReminderSummaryCard`).

---

### Phase 2 — Visual Additions (2-3 weeks, pure CSS/SVG, no new deps)

#### 6. Daily Sales Trend Bars (Week/Month views)
**What:** Vertical bar chart showing sales per day for the selected period.
**Why:** Reveals patterns like "Mondays are busy, Wednesdays are slow."
**Data:** New `computeDailySales(rows)` that groups by calendar day and sums per day.
**Rendering:** Pure CSS `<div>` bars with height proportional to daily total. Day labels underneath. Green (#1B4332) bars on warm paper background.
**Files:** `reportSelectors.js` (new function), `ReportView.jsx` (new section between KPIs and search).

#### 7. Cash-vs-Transfer Donut
**What:** Small 80×80 CSS donut chart using `conic-gradient` showing payment method split.
**Why:** Ethiopian shopkeepers juggle Telebirr, CBE Birr, M-Pesa, and physical cash. Need to see the balance.
**Data:** `metrics.cashExpected` and `metrics.transferRecorded` — already computed.
**Rendering:** Pure CSS `conic-gradient` with white inner circle. Legend dots below.
**Files:** `ReportView.jsx` (new component, placed near the cash/transfer KPI cards).

#### 8. Expense Trend Comparison (Week/Month)
**What:** Side-by-side expense breakdown: current period vs prior period.
**Why:** Shows if spending patterns are changing.
**Data:** Reuses expense breakdown data from Phase 1 + prior-period computation from Phase 2.
**Rendering:** Two stacked horizontal bar groups with category alignment.
**Files:** `ReportView.jsx` (extends existing expense breakdown section).

---

### Phase 3 — Advanced Analytics (3-4 weeks)

#### 9. Profit Margin Trend (SVG Line Chart)
**What:** 12-week average profit margin trend as an SVG polyline.
**Why:** Shows long-term business health trajectory.
**Data:** New `computeMarginTrend(transactions, weeks)` that computes `(profit / amount) * 100` per week.
**Rendering:** SVG `<polyline>` with axis labels, gridlines, green primary color.
**Gating:** Behind `hasAdvancedReports` entitlement.
**Files:** `reportSelectors.js` (new function), `ReportView.jsx` (new section), optional new `reportExport.js`.

#### 10. Shareable Report Export (Image/PDF)
**What:** One-tap export of a styled report summary as printable HTML or image.
**Why:** Merchants need to share reports with partners, accountants, or for tax purposes.
**Data:** KPI grid + expense breakdown + top items + credit health.
**Rendering:** Follow `exportCreditReportPdf` pattern from `customerMetrics.js` — open styled HTML in new tab, trigger `window.print()`. Web Share API fallback.
**Gating:** Behind `hasAdvancedReports` entitlement.
**Files:** New `reportExport.js`, button in export section of `ReportView.jsx`.

## Risks

- **Performance**: Two extra `buildReportRows` calls for prior-period comparison could jank on low-end phones with 10K+ transactions. Mitigate with `useMemo` and skip when `timeRange === 'today'`.
- **Schema gap**: `profit` field may be null/older records. Show "—" instead of zero when profit data is unavailable.
- **Amharic RTL**: Item names are freeform Amharic. Use `direction: 'auto'` on text elements (existing pattern).
- **No chart library**: Deliberate choice. recharts adds ~200KB gzipped. Pure CSS bars + inline SVG lines stay under 5KB total and work offline.
