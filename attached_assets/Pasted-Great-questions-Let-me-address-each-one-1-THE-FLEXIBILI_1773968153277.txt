## 1. THE FLEXIBILITY PROBLEM

You're right. Every shop is different. So the app should have **ZERO assumptions about what they sell.**

```
Bread seller types:
"Bread 10 birr, 100 pieces, Bread 40 birr, 3, total 120"

Shoe seller types:
"Shoe size 41, 2000 birr"

Cloth seller types:
"Cotton cloth 3 meters, 450 birr"

Phone accessories seller:
"Screen protector 5 pieces, 150 each, total 750"
```

### The Key Insight: Don't Over-Engineer Voice

```
What you DON'T need:
─────────────────────
❌ Parse "bread 10 birr x 100" into structured data
❌ Recognize product names automatically
❌ Build a shopping list from voice
❌ Understand complex sentences

What you DO need:
─────────────────
✅ Convert speech to text
✅ Extract the NUMBER (amount) from the text
✅ Let user confirm or edit
✅ Save it
```

### Voice → Text → Simple Parsing

```kotlin
// PSEUDOCODE — Keep it simple

fun onVoiceResult(rawText: String) {
    // Step 1: Show the raw text to user
    textView.text = rawText
    
    // Step 2: Try to find a number in the text
    val amount = extractLargestNumber(rawText)
    
    // Step 3: Pre-fill the amount field
    amountField.setText(amount?.toString() ?: "")
    
    // Step 4: Let user confirm or edit
    // That's it. Nothing fancy.
}

fun extractLargestNumber(text: String): Double? {
    // Look for patterns like:
    // "1000" or "one thousand" or "1,000"
    // Just find the biggest number — that's probably the total
    
    val digitNumbers = Regex("\\d+")
        .findAll(text)
        .mapNotNull { it.value.toDoubleOrNull() }
        .toList()
    
    return digitNumbers.maxOrNull()
}
```

## 3. DO YOU NEED OFFLINE-FIRST FOR MVP?

### Short Answer: Yes. But "offline-first" doesn't mean what you think.

### What "Offline-First" Actually Means for MVP

```
You DON'T need:
────────────────
❌ Cloud sync engine
❌ Conflict resolution
❌ Queue system for pending sync
❌ Firebase real-time database
❌ Multi-device sync

You DO need:
────────────
✅ App works when internet is off
✅ Data stored on the phone
✅ No error screen when connection drops
```

### Why You Can't Skip This

```
Scenario 1: Shop with WiFi
─────────────────────────
Owner has WiFi at shop.
App works. Great.

Scenario 2: Shop with mobile data, sometimes off
─────────────────────────────────────────────────
Owner has mobile data but it cuts out.
Without offline support:
  → App shows "No connection" error
  → Owner can't add entry
  → Owner picks up paper notebook instead
  → Owner never comes back

Scenario 3: No data at all
──────────────────────────
Owner doesn't have data package today.
Without offline support:
  → App is completely useless
  → Paper notebook wins again
```

### The Reality of Ethiopian Internet

```
- Mobile data costs money per MB
- Many shop owners turn data OFF to save money
- They turn it on only for WhatsApp or urgent things
- If your app NEEDS internet = they won't use it
- Paper never shows "No connection" error
```

### MVP Database (Dead Simple)

```kotlin
@Entity(tableName = "entries")
data class Entry(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,
    
    val description: String,    // "Bread 10 birr x 100"
    val amount: Double,         // 1000.0
    val type: String,           // "SALE" or "EXPENSE"
    val timestamp: Long,        // when it was added
    val date: String            // "2026-03-20" for grouping
)

@Dao
interface EntryDao {
    @Insert
    suspend fun insert(entry: Entry)
    
    @Query("SELECT * FROM entries WHERE date = :date ORDER BY timestamp DESC")
    suspend fun getEntriesForDate(date: String): List<Entry>
    
    @Query("SELECT SUM(amount) FROM entries WHERE date = :date AND type = 'SALE'")
    suspend fun getTotalSales(date: String): Double?
    
    @Query("SELECT SUM(amount) FROM entries WHERE date = :date AND type = 'EXPENSE'")
    suspend fun getTotalExpenses(date: String): Double?
    
    @Query("SELECT DISTINCT date FROM entries ORDER BY date DESC")
    suspend fun getAllDates(): List<String>
}
```

**That's the entire data model for MVP. One table. Five fields. Done.**

## LEARNING QUESTIONS AFTER MVP LAUNCH

After 2 weeks with 20 users, answer these:

```
1. Do they open the app daily?
   YES → Core habit is forming
   NO  → Why? Friction? Forgetfulness? Paper is easier?

2. How many entries per day?
   HIGH → App is useful
   LOW  → They're not finding it valuable

3. Voice vs Text ratio?
   MOSTLY VOICE → Invest in better voice
   MOSTLY TEXT  → Voice isn't solving a problem

4. What do they ask for?
   → Build THAT next

5. Do they go back to paper?
   YES → App is harder than paper. Fix the friction.
   NO  → You're winning
```

**Every next feature comes from these answers, not from your brainstorm.**
