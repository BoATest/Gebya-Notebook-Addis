### P2.4 Security: restoreFromJSON Auth Token Wipe
**File:** `src/utils/restore.ts` or wherever restoreFromJSON logic resides

**Problem:** Critical security issue - backup restoration wipes all authentication tokens

**Solution:** Preserve auth tokens during restore
```javascript
// In restore function:
// Save current auth tokens before restore
const savedTokens = {
  authToken: await getAuthToken(),
  refreshToken: await getRefreshToken()
};

// Perform restore
await performRestore(data);

// Restore auth tokens after restore
await storeAuthToken(savedTokens.authToken);
await storeRefreshToken(savedTokens.refreshToken);
```

**Impact:** Security - prevents accidental auth token loss
**Priority:** High (Security)
**Effort:** Medium

### P2.5 Replace window.confirm with Custom Modal
**File:** `src/components/settings/PaymentChannelsSection.jsx`

**Problem:** Uses browser native window.confirm instead of app's custom modal

**Solution:** Import and use ConfirmDialog component
```javascript
// Replace:

if (!await ConfirmDialog.show(
  lang === 'am' ? 'ይህን መንገድ ይሰርዙ?' : 'Remove this channel?'
)) return;
```

**Impact:** Consistent app UI, better user experience
**Priority:** Medium
**Effort:** Low

### P2.6 Fix Theme Context Race Condition
**File:** `src/context/ThemeContext.jsx`

**Problem:** ThemeContext flashes light→dark on page load due to localStorage vs IndexedDB race

**Solution:** Ensure consistent theme initialization
```javascript
// In ThemeContext, use single source of truth
const [theme, setTheme] = useState(() => {
  // Try localStorage first, fallback to IndexedDB
  const stored = localStorage.getItem('theme');
  if (stored) return stored;
  
  // Synchronize with IndexedDB
  return db.settings.get('theme_preference')
    .then(row => row?.value || 'light')
    .catch(() => 'light');
});
```

**Impact:** Consistent theme experience across session
**Priority:** Medium
**Effort:** Medium

### P2.7 Fix PWA Install Strip Layout
**File:** `src/components/SettingsPage.jsx`

**Problem:** Install strip uses hardcoded maxWidth: 390px, off-center on wider viewports

**Solution:** Use responsive design instead of fixed width
```javascript
// In SettingsPage.jsx install strip:
<div className="fixed flex items-center gap-3 px-4 py-3 z-50"
  style={{ /* Responsive positioning */
    left: '50%',
    transform: 'translateX(-50%)',
    bottom: 56,
    width: '100%',
    maxWidth: '100%', // Remove fixed constraint
    background: 'rgba(250,249,244,0.95)',
    backdropFilter: 'blur(10px)',
    borderTop: '1px solid rgba(27,67,50,0.08)',
  }}
>
```

**Impact:** Better responsiveness on tablet/desktop viewports
**Priority:** Low
**Effort:** Low

### P2.8 Fix Date.now() ID Collision Risk
**File:** `src/components/settings/RecurringExpensesPanel.jsx`

**Problem:** Uses Date.now() as item ID, collision risk for two items added in same ms

**Solution:** Use deterministic ID generation
```javascript
// Replace:
const id = Date.now() + Math.random().toString(36).substr(2, 9);

// With:
const id = `recurring-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
```

**Impact:** Eliminates ID collision possibility
**Priority:** Medium
**Effort:** Low
